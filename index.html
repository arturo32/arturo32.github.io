Triangle Lerp

This one I've taken from a Twitter thread by Freya Holm√©r. I was used to the idea of linear interpolation for the Drawing Project, but I had never heard of interpolating between three points at the time.

Julia sets

Move your mouse to generate beautiful products of a complex number function.

Julia sets

The Mandelbrot set is a set of complex numbers that look pretty when plotted in a complex plane like the one below:

To generate this is relative simple. It all boils down to the function below:

\(f_c(z) = z^2 + c\)

We will talk about the both complex numbers \(z\) and \(c\) soon. First, we need another pair of numbers:

  • A positive number greater than 2. Let's call it the threshold;
  • Another positive number. This time a larger one. This will be our number of iterations.

Now, we apply our function recursively to each pixel of an image representing the complex plane. We do this by making \(c\) the coordinate of the current pixel and \(z\) starting with the value \(0 + 0i\). As we iterate, \(z\) may or may not grow beyond our threshold. If it gets there before we reach the number of iterations, we stop the recursion and paint the pixel white. If not, we paint it black. That's it!

And Julia sets are almost the same thing, with \(c\) and \(z\) switching places, i.e., \(z\) will represent a pixel coordinate while \(c\) will be... an arbitrary constant! For each \(c\) we choose at the start of our process, we can make entirely different sets. Each of them possessing a unique complexity and beauty. And for making things even prettier we can add a little bit of math magic to the RGB values, changing them by how fast some of the inputs can reach our threshold, like the image below made with \(c = 0 - 0.8i\):

Finally, the GIF of the thumbnail of this project was made by changing \(c\) as the coordinates of the cursor. So at each new position of the mouse the function iterated thought all the pixels to render the image for that particular set. To save the processor, I made the function skip some pixels. This is why the colors are a bit washed out.

Times table

Wonderful patterns emerge when numbers are written in a circle.

Times table

...times table? That's right! The GIF in the thumbnail is simply a 2 times table, where each point on the circumference is a number and each line connects a number to its double. Thus, 2 connects to 4, 3 connects to 6, and so on. In the image below we only have 12 numbers. So, from 6 and beyond we do what's called clock arithmetic, or more precisely, modular arithmetic. 0 is 12, 1 is 13, 2 is 14, and so on... This relation between the numbers is called congruence. That's why 6 connects to 0 and 7 connects to 2. We can called it the 2 time table module 12:

How Pointers Work

Personal educational project about the magic of pointers and what are they like in memory.

How Pointers Work

When I saw my colleagues struggling with the concepts of pointers in introductory programming classes, I thought of writing a program that could help them visualize what happens in the computer's memory when they run their programs.

So 2 years ago I started making a website that shows memory allocation of variables and their pointers as they are typed into a console.

As of now it can identify declaration and assignment of variables and pointers and their types, as well as dereferencing of pointers. It cannot yet identify arrays (which are sort of pointers, important ones!). I plan to use Vue.js to manage the visualization of the memory cells instead of p5.js and to finish it this year.

I talk more about it in the project's page below.

Ray Tracer

3D ray tracer that can create photorealistic images.

Ray Tracer

This project was developed by me and my friend Jo√£o V√≠tor throughout the course of Computer Graphics, taught by professor Selan Rodrigues dos Santos. It creates images representing 3D scenes described by XML files. All of this using the magic of ray tracing.

Ray tracing is a technique used to create 3D images where photo-realism is desired. It works by simulating photons of light as rays (vectors) that are shot at the scene from the viewer's perspective. The rays then travel through the scene being reflected, refracted or absorbed by objects. All of these interactions are calculated for each pixel of the image, defining its final color.

Doing this project was challenging because of its size. We had to take into account the camera, objects and their interactions with rays, different types of lights, different light models, parsing XML files and .obj files... At the same time it was very rewarding because at each step we could see the result of what we had done at that point.

In the end, we couldn't believe the images that our own program made! The (optional) spherical background made the reflective surfaces much more interesting!
